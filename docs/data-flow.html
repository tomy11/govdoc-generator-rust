Endpoint Map

govdoc-api
GET /status

รายงาน backend ที่ใช้งานอยู่: LLM, embedding, OCR, renderer, persistence

POST /generate

รับ `DocRequest`, ดึง examples, เรียก LLM, ส่งกลับ document JSON + trace

POST /edit

แก้ document JSON ตาม instruction และ optional target fields

POST /render

resolve template แล้วส่งข้อมูลให้ renderer เพื่อคืนไฟล์ `.docx`

POST /ingest

เก็บ structured example ลง memory และเพิ่ม vector ถ้ามี embedding

POST /ingest/ocr/upload

upload รูป/PDF, OCR, structure ด้วย LLM, เก็บเป็นตัวอย่างให้ retrieval

GET /templates

list template ตาม `doc_type` หรือ `agency`

POST /templates/upload

บันทึก `.docx` ลง `GOVDOC_TEMPLATES_DIR` และ register ใน SQLite

GET /templates/default

หา default template ตามประเภทเอกสารและหน่วยงาน

POST /documents

บันทึกผลลัพธ์ generated document ลง SQLite

GET /documents/:id

เปิดเอกสารที่เคยบันทึกไว้กลับมาแสดงใน UI

DELETE /documents/:id

ลบ saved document ออกจาก SQLite

Runtime Data

ignored from git

SQLite

  • `gov_doc_memory` สำหรับ retrieval examples
  • `doc_template` สำหรับ metadata ของ template
  • `document` สำหรับเอกสารที่บันทึกไว้

Vector Index

  • cache จาก embeddings ใน SQLite
  • rebuild ได้เมื่อไฟล์หาย
  • ใช้ค้น examples ที่ใกล้เคียงตอน generate

Templates

  • ไฟล์ `.docx` ที่ user upload
  • เก็บใต้ `GOVDOC_TEMPLATES_DIR`
  • ใช้ตอน `POST /render`
